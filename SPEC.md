# Tool-Null Confabulation Probe — Project Spec

## What we are building
A linear probe on residual-stream activations that predicts, given a tool
returned an empty result, whether the model is about to fabricate a numerical
answer or honestly admit no data was found. Plus an inference-time
intervention that uses the probe to reduce fabrication.

## Why
Autonomous research agents fabricate plausible numbers when their tools
return empty. This contaminates downstream experimental decisions. Generic
hallucination probes underperform here because their training distribution
does not include tool-call structure with explicit empty results in context.
A probe trained on the specific (post-tool-call, about-to-generate)
distribution should outperform.

## The exact target
- Inputs: residual stream activations at the final prompt position, before
  generation, for a Llama-3.1-8B-Instruct agent that has just received an
  empty tool result.
- Labels: FABRICATE (1) or ADMIT (0), assigned by a Claude judge calibrated
  against 50 hand labels.
- Trained on empty-result trajectories only. The contrast is intra-empty,
  not data-vs-empty.

## Model and seeds
- Model: meta-llama/Llama-3.1-8B-Instruct, pinned to a specific revision
  recorded in config.yaml.
- Tool: Materials Project API (mp-api), no-auth, free.
- Seeds: torch=42, numpy=42, random=42, set at the top of every script.
- Precision: bf16 on A100. If T4, document quantization and re-run probe
  evaluation to check for degradation.

## Data
- 200 real materials from Materials Project with band_gap,
  formation_energy_per_atom, and density all populated.
- 200 plausible perturbations, each verified to return empty from the API.
- 5 query templates per property, written before trajectory collection
  begins and frozen.
- 3 system prompts on the empty side: neutral, pressure-to-estimate,
  honesty-cued. Goal: produce both FABRICATE and ADMIT labels at meaningful
  rates (target: neither below 20%).

## Splits
- Train/test split is BY MATERIAL, not by trajectory.
- The intersection of train and test material sets must be empty. This is
  enforced by a runtime assertion, not a comment.
- Cross-chemistry generalization split: train on oxides, test on sulfides
  (or analogous), reported as a separate number.

## Activation extraction
- Position: last token of the templated prompt, before any generation
  tokens are produced.
- For 5 random trajectories, the decoded token at the extraction position
  is printed and inspected manually. It should be the assistant-turn-start
  token, not anything generated by the model.
- All 33 layers stored. Layer sweep is part of the result, not a hyperparam.

## Labeling
- Judge: Claude, with a fixed prompt template stored in prompts/judge.md.
- Hand-label 50 trajectories before judge labeling. Compute agreement.
- If judge-vs-hand agreement < 85%, refine judge prompt and rerun on the
  same 50 until calibrated. Calibration trajectories are then excluded from
  training to avoid contamination.
- AMBIGUOUS labels are dropped from probe training and reported as a
  fraction of total.

## Probe
- Architecture: logistic regression, sklearn, C=1.0, max_iter=1000.
- Features: raw residual stream activation at a given layer.
- Trained per-layer; reported metric is AUROC peak across layers, with the
  full curve shown.

## Required baselines (probe must beat these or we report an honest negative)
1. Trivial regex: does the tool output contain "[]" or "not found"?
2. Next-token entropy of the model on the empty-side trajectories.
3. Self-ask: append "Do you have sufficient information to answer?" and
   parse a yes/no.
4. Off-the-shelf-style probe: same architecture, trained on TruthfulQA-style
   data, applied zero-shot.

## Intervention
- Threshold τ chosen on a validation split; not tuned on test.
- Two variants:
  - Prompt-injection: when probe fires, prepend a system message before
    generation noting the tool returned no data.
  - Activation-space: subtract α × (normalized probe direction) from the
    residual stream at the firing layer at the last position. Sweep α.
- Reported metrics:
  - Fabrication rate on held-out empties, with and without intervention.
  - Accuracy preservation on data-present queries (must not degrade by
    more than 5 absolute points).
  - False-positive rate of the probe on data-present queries.

## Transfer
- Cross-tool: train on Materials Project, test on PubChem (or a synthetic
  tool with deliberately different schema and naming).
- Cross-template: held-out paraphrased query templates not seen in training.

## Success criteria
- Probe peak AUROC > 0.75 on the empty-only, material-split test set.
- Probe beats all four baselines on the same test set.
- Intervention reduces fabrication rate by ≥ 30% relative on held-out
  empties.
- Cross-tool transfer AUROC > 0.65.

## Invalidation conditions (we treat the result as broken if any of these hold)
- Trivial regex baseline achieves AUROC > 0.95 on test set.
- Train and test materials overlap.
- Layer-sweep curve is flat or monotonically increasing across all 33 layers.
- Judge-vs-hand agreement on the 50-example calibration set < 85%.
- FABRICATE or ADMIT label is < 5% of empty-side trajectories (no signal
  to learn).
- Probe AUROC on data-present-vs-empty trivially exceeds AUROC on
  fabricate-vs-admit, indicating the probe learned the wrong contrast.

## Stage gates (no stage proceeds without explicit human approval)
1. Data construction → human reviews 20 perturbations and 20 query/template
   pairs.
2. Trajectory collection → human reads 30 random trajectories, confirms
   fabricate-vs-admit balance is non-degenerate.
3. Labeling → human hand-labels 50, judge agreement computed and reported.
4. Probe training and baselines → human reviews layer sweep, baseline
   comparison table, and the printed train/test material disjointness check.
5. Intervention → human reviews fabrication-rate-down and quality-preserved
   numbers together.
6. Transfer and ablations.

## Things the human does, not the agent
- Hand-labeling 50 trajectories.
- Reading 30 random trajectories before authorizing labeling.
- Writing the final results table and the writeup.

## Things that must be runtime assertions, not comments
- Train and test material sets are disjoint.
- Activation extraction position decodes to the expected token.
- Judge agreement on the calibration set passes the 85% threshold.
- Label distribution on training set is computed and printed.
